h1. Data Stuctures fo Mydvds

h2. Users

*Type:* HASH
*Keyname:* user:<userid>

h3. Fields

* email (string)
* firstname (string)
* lastname (string)
* joinDate (date)
* lastLogin (datetime)
* status (string enum: pending, confirmed, subscriber)
* referredBy (userid)


h2. Dvds

*Type:* HASH
*Keyname:* dvd:<dvdid>

h3. Fields

* title (string)
* barcode (string)
* status (string enum: public, private, pending)
* addedBy (userid)
* addedOn (datetime)
* modifiedOn (datetime)


h2. Loans

*Type:* HASH
*Keyname:* loan:<loanid>

Each item in SET is ID of a Loan HASH.

h3. Fields

* lendee ID (userid)
* lend_date  (datetime)
* return_date  (datetime)
* list_item_id  (itemid)
* lender_id (userid)
* lender_email (email)

h3. Users loans

*Type:* SET
*Keyname:* loans:<userid>
